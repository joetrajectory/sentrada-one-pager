# The grounding-gate exam (runner/cases/)

A committed library of cases the `harness` command runs the CURRENT 4b template
over: one grounding call per case, nothing else in the chain. Every incident
that shipped (or nearly shipped) a factual error lives here as a named case, so
a template edit that quietly un-learns an old lesson shows up as a named
regression on the scorecard. Scope is factual and credibility errors only:
fabricated facts, wrong numbers, invented places, invented or misattributed
quotes, stripped qualifiers. No tone rules, no style rules, no placement rules
(those belong to ship-check's copy lint and gate-probe's lint probes).

## Commands

```bash
python runner/sentrada_runner.py harness                  # full run + scorecard
python runner/sentrada_runner.py harness --only <case-id> # spot check, no save
python runner/sentrada_runner.py flag --slug <piece> --note '... "offending phrase" ...'
python runner/sentrada_runner.py retro                    # flags -> rules, with approval
```

`batch-build` warns (and asks for an explicit yes) when any gate template file
changed after the last saved harness run.

## Case format

One directory per case:

- `research.md` — the research the copy is checked against, verbatim.
- `copy.txt` — the gate-visible copy text, built by the same copy-text builder
  production uses (edition line included for newspapers).
- `expected.json` — `{"verdict": "pass"}` or
  `{"verdict": "must_flag", "violation_type": ..., "offending_detail": ...,
  "match_terms": [...]}`. A must-flag counts as CAUGHT only when the gate flags
  it AND a match term appears in the gate's issues; flagged-for-a-different-
  claim counts as a MISS, because the real error still ships. `"draft": true`
  marks a case created by `flag` and not yet confirmed at a retro.
- `sender_facts.txt` — email cases only; the sender facts the piece was gated
  under.
- `note.md` — provenance.

Keep the whole set under 25 cases so a full run stays cheap (one Opus call per
case). If the library needs to grow past that, retire the least-load-bearing
poisoned case rather than letting runs bloat.

## Provenance of the seed set (July 2026)

- 4 `clean-*` must-pass cases: real shipped/proofed copy, verbatim, from
  `newspaper/cognism.json`, `newspaper/mmc.json`, `crossword/test_cognism.json`
  and `email/test_qflow.json`. Their research docs were authored for the exam
  to carry every claim the copy makes (the original research lived in ephemeral
  piece folders); the research is the ground truth a case is scored against, so
  treat it as part of the case, not as company fact.
- 6 `history-*` must-flag cases reconstruct the documented incidents: the
  stripped "over" qualifier and the fabricated edition-line city (Sainsbury's
  piece, 6 July 2026 retro), the template exemplar cities copied under gate
  pressure (batch 2026-07-09 retro), the Project Pinata "nearly" attachment
  (batch 2026-06-25 retro), and the unsourced sub-1% reply-rate commonplace and
  elapsed-time dateline arithmetic (cold-review catches, 13 July 2026).
- 5 `poisoned-*` must-flag cases: real copy with exactly one fact broken each,
  authored directly (never generated by the chain), covering a wrong number, an
  invented place, a misattributed quote, a wrong date and an unsupported sender
  claim.

## What lives beside this

`scorecard.json` — the last full harness run (plus compact history): per-case
results and a hash of every template file the gate loads, which is what the
batch staleness guard checks. `retro_log.md` — every `flag` appends an entry;
every `retro` closes the entries above it with a marker and the paste-ready
Notion changelog lines for applied rules. Commit all of it: containers are
ephemeral, the exam is not.
